# Trabalho Alavancado

*(leveraged work)*

Esforços que multiplicam o que times podem alcançar muito depois de você ter seguido em frente. Contraste com trabalho direto, onde o impacto é proporcional ao tempo investido e para quando você para.

## A ideia central

Trabalho alavancado não é fazer mais — é fazer o tipo certo de coisa. Uma decisão de arquitetura bem documentada pode guiar o time por anos. Um engineer bem mentorado pode repassar o que aprendeu para outros dez. Um processo de code review bem calibrado pode elevar o piso de qualidade do time inteiro sem esforço contínuo.

A alavancagem é o multiplicador: o mesmo número de horas produz impacto radicalmente diferente dependendo de onde é aplicado.

## Aplicação no papel de tech lead

Dentro do papel de [[tecnico/concepts/tech-lead]], trabalho alavancado é a razão pela qual o objetivo final é tornar-se desnecessário no cargo atual. Se você está insubstituível, criou dependência — o oposto de alavancagem.

Exemplos de trabalho alavancado para um tech lead:
- Documentar o *porquê* de decisões arquiteturais, não só o *o quê*
- Criar um ambiente onde engenheiros resolvem problemas sem você na sala
- Estabelecer padrões de qualidade que o time aplica autonomamente
- Mentorear um engineer júnior que passa a mentorear outros

## Contraste com trabalho essencial

Ser "essencial" (insubstituível) parece valoroso, mas é uma armadilha. Pessoas essenciais são gargalos — o time só avança tão rápido quanto o gargalo permite. Trabalho alavancado tem o objetivo oposto: maximizar o que acontece *independentemente* de você.
